# 레이아웃(navbar, footer)
- [ ] 일반 회원가입 페이지 진입
- [ ] 관리자 회원가입 : 진입 시 비밀번호 입력하게 하는 화면 연결(개선필요)
- [ ] 로그인 중인 회원 상태 표시
- [ ] 각 메뉴별 페이지 이동 상태
- [ ] 네비게이션 바 반응형 정상작동(화면 줄였을 시 로고, 메뉴 나타남 현황, 메뉴 배경 흰색)
- [ ] footer 구성내용 확인(사진, 이름, 이메일, 역할)
- [ ] 리더기 버튼 누르면 배포한 프론트 북리더로 이동

# 메인페이지
- [ ] 메인페이지 신간 5개씩 출력
- [ ] 메인페이지 베스트셀러 5개씩 출력
- [ ] 네비 바 검색 시 검색내용 20개 출력

# 회원가입 
- [ ] 일반 회원가입, 관리자 회원가입, 뒤로가기 버튼
- [ ] 일반 회원가입 - 인증용 이메일 발송
- [ ] 일반 회원가입 - 인증번호 일치
- [ ] 일반 회원가입 - 비밀번호 확인
- [x] 모든 항목 입력
- [ ] 관리자 회원가입 인증 키 정상작동
- [ ] 관리자 인증 키 input type password로 변경
- [ ] 가입 시 환영인사(반갑습니다 북유럽입니다)
- [ ] 아이디 비번 찾기 메뉴 제거

# 로그인페이지
- [ ] 구글 소셜계정 로그인 / 로그아웃
- [ ] 네이버 소셜계정 로그인 / 로그아웃
- [ ] 카카오 소셜계정 로그인 / 로그아웃
- [ ] 이메일 계정 로그인 / 로그아웃
- [ ] 아이디 찾기 - LastName, FirstName, Phone 정보 일치하면 아이디 다 보여줌
- [ ] 비밀번호 찾기 - 아이디, LastName, FirstName, Phone 정보 일치 하면 등록된 이메일로 임시비번 발송


# 도서 등록
- [ ] 커버이미지 드래그 앤 드랍으로 등록이 되어야 한다
- [ ] PDF파일 드래그 앤 드랍으로 등록이 되어야 한다
- [ ] 제목, 작가, 출판사, 출간일, 정가, 할인가 상품설명, 카테고리, 청불여부, ISBN이 등록되어야 한다
- [ ] 상품정보 및 내용 > 도서정보 및 내용으로 변경
- [ ] 상품 설명 > 도서 설명 으로 변경
- [ ] ISBN '자동 생성됩니다' 추가
- [ ] ISBN 라벨 > BOOK_U_LUV 자체 도서ID(ISBN대체로 변경)
- [ ] 등록 완료 시 등록이 완료되었습니다 라는 알림 출력
- [ ] 등록 완료 시 해당 도서의 상세페이지로 이동


# 도서 수정
- [ ] 도서등록사항 + 아무것도 변경하지 않아도 수정이 되어야 한다.
- [ ] 도서 수정 시 기존 정보가 세팅이 되어있어야 한다.
- [ ] 수정하면 수정이 완료되었습니다 라는 알림 출력
- [ ] 수정 완료 시 해당 도서의 상세 페이지로 이동

- [ ] 도서 목록이 20개씩 출력되고 페이징이 가능해야 함
- [ ] 도서 통합검색이 가능해야 함
- [ ] 키워드 별 검색결과가 제공되며 출력결과의 건수를 표시함.
- [ ] 커버이미지 + 제목까지 클릭 시 도서 상세보기 페이지로 이동

- [ ] 도서 목록이 20개씩 출력되고 페이징이 가능해야 함
- [ ] 도서 통합검색이 가능해야 함

# 도서상세페이지
- [ ] 커버이미지가 출력되어야 함
- [ ] 도서의 모든 항목이 출력되어야 함
- [ ] 관리자계정만 수정 / 삭제 버튼이 보여야 함
- [ ] 도서 수정페이지로 이동이 가능해야 함
- [ ] 도서 수정 시 컨펌해야 함.
- [ ] 도서 삭제가 가능해야 함.
- [ ] 도서 삭제 시 정말 삭제할 건지 컨펌해야 함.
- [ ] 컨펌 완료하면 도서 삭제가 완료되었다는 알림 출력
- [ ] 현재평점, 총평점 밑으로 수치가 표시되어야 함(예시. 3.75 / 5.0)
- [ ] 별점을 동반한 리뷰 등록이 가능해야 함.
- [ ] 리뷰 등록 시 등록되었다는 문구가 생겨야 함.
- [ ] 등록한 리뷰가 보여야 함.
- [ ] 등록한 리뷰를 작성자만 수정할 수 있어야 함.
- [ ] 등록한 리뷰 수정 창에서 취소가 가능해야 함.
- [ ] 등록한 리뷰를 작성자만 삭제할 수 있어야 함.
- [ ] 장바구니에 해당 도서를 담았을 시 장바구니에 담겨야 함.
- [ ] 장바구니에 해당 도서를 담았을 시 장바구니로 이동할 지 컨펌하고 확인 버튼을 누르면 이동해야 함.
- [ ] 구매버튼을 누르면 해당 주문 상세정보가 표시되고 결제를 누르면 결제가 진행되어야 함.
- [ ] 주문 상세정보 페이지에서 취소를 누르면 이전 페이지로 돌아가야 함.
- [ ] 결제과정에서 취소를 누르면 유저가 취소했습니다. 라는 알림을 띄움.


# 공지사항 등록페이지

- [ ] 등록 완료 시 등록이 완료되었다는 알림이 뜨고 목록 페이지로 이동해야 함.
- [ ] 관리자만 수정 / 삭제 버튼이 보이고 수정 페이지로 이동이 가능해야 함.
- [ ] 공지사항 삭제가 가능해야 함.


# 공지사항 상세페이지
- [ ] 관리자만 수정 / 삭제 버튼이 보이고 수정 및 삭제가 가능해야 함.
- [ ] 공지 수정 시 기존 데이터가 수정 폼에 세팅되어야 함.
- [ ] 수정이 완료되면 수정이 완료되었다고 알림이 떠야 함.
- [ ] 삭제할 건지 물어보고 삭제된 공지사항은 되돌릴 수 없다고 컨펌이 떠야 함.
- [ ] 삭제가 완료되면 삭제가 완료되었다고 알림이 떠야 함.
- [ ] 제목, 내용, 작성일, 수정일, 조회수가 출력되어야 함.
- [ ] 수정내용이 존재할 때만 수정일시가 업데이트 되어야 함.


# 공지사항 수정페이지
- [ ] 기존 제목과 내용이 세팅되어있어야 함.
- [ ] 수정 버튼을 누르면 수정이 진행되고 수정이 완료되었다고 알림이 떠야 함.
